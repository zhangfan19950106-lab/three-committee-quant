## ⚙️ L1 — 业务规则

### 架构：三委员会量化（V5，2026-05-15）
- **三阶段串联**：研究委员会（4人并行，只给事实）→ 风险委员会（3人并行，读研究结果）→ 投决会（3人并行，读研究+风险）
- **辩论精简**：仅多头vs空头→仲裁员，风控复用第二阶段输出，不再重复调用API
- **角色分配**：研究委→技术/基本面/情绪/新闻；风险委→激进/保守/中性风控；投决会→多头/空头/仲裁员
- **仲裁员替换原组合经理**，终裁全权负责
- **名称**：`run_analysis.py` 总指挥 + `fetch_template.py` 三委员会 + `run_xiaomi_debate.py` 投决会辩论

### 运行规则
- **9Agent标准顺序**：研究委(技术/基本面/情绪/新闻)→风险委(激进/保守/中性)→投决会(多头/空头/仲裁员)（05-15）
- **评分语义**：高分→倾向买入（统一口径，05-12）
- **NeoData标准六件套**：行情/技术/资金/利润/资产/现金流（04-29）
- **Context构建优先级**：最新行情→资金流向→技术面→日线→财务→研报→新闻（05-09）
- **模型路由**：deepseek-chat（日常分析，并行调用），deepseek-reasoner（辩论终裁，04-29）
- **监工Agent**：审查-打回-再审循环，MAX_RETRIES=2（05-12）
- **报告归档**：`C:\Users\zhang\Desktop\TradingAgent报告存放\股票名_YYYYMMDD\`（04-29）
- **手续费感知**：脚本支持 POSITION/FEE_PER_TRADE 参数，终裁自动算摩擦成本（05-12）
- **单命令全流程**：`run_analysis.py --stock X --code Y --cost Z`，内部走fetch→分析→辩论→报告
- **总耗时**：三阶段~1.5min（研究~55s + 风险~55s + 投决会~60s + 辩论~95s）
